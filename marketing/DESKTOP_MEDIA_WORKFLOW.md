# Desktop Media Save Workflow

All ThermaForge content/media created moving forward should be saved into this local Desktop command-center folder first:

```text
~/Desktop/ThermaForge-Media-Command-Center
```

In this environment that resolves to:

```text
/root/Desktop/ThermaForge-Media-Command-Center
```

## Folder map

| Desktop folder | Use |
|---|---|
| `00_README_START_HERE/` | Handoff notes and folder instructions. |
| `01_Strategy_and_Calendar/` | Launch plans, calendars, UTM patterns, platform notes. |
| `02_Organic_Posts/` | Organic copy banks for X, LinkedIn, Instagram, Facebook, Threads, Google Business Profile, and community platforms. |
| `03_Paid_Ads/` | Paid ad copy banks, audiences, creative briefs, and landing-page variants. |
| `04_Visual_Concepts_and_Prompts/` | Visual prompts, shot lists, carousel concepts, video scripts, and production notes. |
| `05_Ready_To_Post/` | Finished posts, graphics, videos, and captions ready to publish. |
| `06_Source_Brand_Assets/` | TF logos, wordmarks, icons, and source brand files. |
| `07_Exports_By_Platform/` | Final exports separated by platform: X, LinkedIn, Instagram, Facebook, Threads, TikTok/Reels/Shorts, Google Business Profile, Nextdoor, Email/SMS. |
| `08_Published_and_Used/` | Assets that have already been posted or used in ads. |
| `09_Archive/` | Old versions, unused drafts, and dated bundles. |

## Naming convention

Use this pattern for all new media files:

```text
YYYY-MM-DD_platform_theme_assettype_version.ext
```

Examples:

```text
2026-05-08_linkedin_missed-calls_carousel_v01.png
2026-05-08_instagram_first-call-recorded_reel-script_v01.md
2026-05-08_meta_dispatch-leak_paid-ad_v01.csv
```

## Going-forward rule

When new ThermaForge content is created:

1. Save the source/draft into the matching numbered Desktop folder.
2. Save finished media into `05_Ready_To_Post/`.
3. Copy final platform-specific exports into `07_Exports_By_Platform/<platform>/`.
4. After publishing, move or copy the exact posted asset into `08_Published_and_Used/`.
5. Keep repo source files in `marketing/` when the content needs to be version-controlled.
